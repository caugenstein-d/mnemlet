## Task 1: GitHub Actions CI workflow

**Files:**
- Create: `.github/workflows/test.yml`

- [ ] **Step 1: Create the workflow file**

Write `.github/workflows/test.yml` with:

```yaml
name: tests

on:
  push:
    branches: [master]
  pull_request:
    branches: [master]

jobs:
  pytest:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - name: Install
        run: |
          python -m pip install --upgrade pip
          pip install -e ".[dev]"
      - name: Test
        run: pytest -q
```

Do not add: matrix, caching, codecov, lint job. P2 if ever.

- [ ] **Step 2: Local sanity**

The workflow cannot be executed locally without `act`. The local
check is: YAML parses, the install command would succeed.

```bash
.venv/bin/python -c "import yaml,sys; yaml.safe_load(open('.github/workflows/test.yml')); print('yaml ok')"
.venv/bin/pip install -e ".[dev]" --dry-run 2>&1 | tail -5
```

The `--dry-run` may not work on all pip versions; if it errors,
ignore that line — the install command is the same one used locally
during dev, so a healthy `.venv` already proves it works.

- [ ] **Step 3: Stop and report**

Report: file created, YAML parses, `.[dev]` install plausible.
Note that the first real CI run only happens after push. Do not
commit yet.

- [ ] **Step 1: Bug report form**

`.github/ISSUE_TEMPLATE/bug_report.yml`:

```yaml
name: Bug report
description: Something is wrong with Mnémlet.
labels: ["bug"]
body:
  - type: textarea
    id: what-happened
    attributes:
      label: What happened?
      description: A clear, short description of the bug.
    validations:
      required: true
  - type: textarea
    id: reproduce
    attributes:
      label: How to reproduce
      description: Commands, requests, or configuration that triggers the bug.
      render: bash
    validations:
      required: true
  - type: input
    id: version
    attributes:
      label: Mnémlet version
      description: Output of `pip show mnemlet` or the git commit SHA.
    validations:
      required: true
  - type: input
    id: env
    attributes:
      label: Environment
      description: OS, Python version, hardware (e.g. Raspberry Pi 5 16 GB).
    validations:
      required: true
  - type: textarea
    id: logs
    attributes:
      label: Logs or stack trace
      render: text
```
